Overview

Appcrafted assets are NoSQL key-value stores that also handle content management and delivery. Developers can deliver files, images, and data to their mobile or web applications. Applications can also programmatically create, read, update, and delete content and data.

Currently, Appcrafted supports a Web API, a Unity Plugin, and a JavaScript plugin.

We are also working on several other backend services for app developers. We're very interested in feedback and suggestions, and we encourage you to email us or contact us using this form.

Web API

{{showAPIDocs && 'Hide' || 'Show'}}

Note: The Web API documentation is being actively updated.

Before you start

Take note of your Access Key and Secret Key from the Appcrafted Credentials section of your account, as well as the ID of the asset container you wish to integrate into your application.

Authentication

You must authenticate each request to the Appcrafted Web API. You do so by setting the Authorization header on each of your requests.


Note: Request signing is not enabled for the pre-release preview (v0).

Instead, set the Authorization header to:

"Basic " + Base64Encoded(<AccessKey> + ":")

Note that Base64Encoded means that you must Base64-encode the result of concatenating your Access Key with ":". Also note the extra space after "Basic".


The Authorization header consists of an access key and a signature, which is computed from your request and secret key using HMAC-SHA1


Compute the signature:

First, compute the string to sign with your secret key. The string to sign is composed of the HTTP verb (GET, POST, etc.), the value of the Host header in lowercase, and the request URL.

StringToSign = HTTPVerb + "\n" +
              ValueOfHostHeaderInLowercase + "\n" +
              HTTPRequestURL;

Calculate an RFC 2104-compliant HMAC with the string you just created, your Secret Key as the key, and SHA1 as the hash algorithm. For more information, go to http://www.ietf.org/rfc/rfc2104.txt. This is your request Signature.


Set the authorization header:

Create the following string.

HeaderString = "<AccessKey>" + ":" + <Signature from above>

Base64-encode the HeaderString from the previous step.

Set the Authorization header to be the following string:

"Basic " + <String from previous step>

Possible errors:

If the Authorization header is incorrectly formatted, the Appcrafted server will return HTTP 403:

{"message": "Missing or Incorrect Authorization Header"}

If your Access Key is incorrect, the Appcrafted server will return HTTP 403:

{"message": "Unrecognized Access Key"}

If the Signature is incorrectly computed, the Appcrafted server will return HTTP 403:

{"message": "Signature Not Verified"}

Endpoint

http://api.appcrafted.com/

HTTPS may also be used

Response

The server will response with JSON.

Create an Asset

POST /v0/assets/<ContainerID>

Creates a new asset in the specified container.


Arguments (as JSON data):

{
  <Attribute_Name>: {
    "Type": <Type>,
    "Value": <Value>
  },
  "AssetID": <Asset_ID>
}

string Attribute_Name

The name of the attribute.

string Type

The attribute type. 

"STRING", "NUMBER", "STRING_ARRAY", "NUMBER_ARRAY", "URL"

Value (depends on the Type)

string Asset_ID (optional)

An optional ID for the asset. If none is provided, one will be generated automatically.


Response:

The Appcrafted server will respond with HTTP 201 and JSON.

{
  "message": "New Asset Created",
  "ContainerID": "ABC",
  "AssetID": "XYZ"
}

cURL example:

$ curl http://api.appcrafted.com/v0/assets/<ContainerID> \
  -X POST -H 'Content-Type: application/json' \
  -d '{"caption": {"Type": "STRING", "Value": "Hello, world"}}' \
  -u <AccessKey>:

Upload a File

POST /v0/assets/<ContainerID>/<AssetID>/upload_file/<AttrName>

Creates an asset attribute with type FILE containing a link to the file hosted on the Appcrafted CDN.


Parameters:

ContainerID

The ID of the container.

AssetID

The ID of the asset within the container.

AttrName

The name of the file attribute. Must contain only URL-legal characters.


Request body:

The file to be uploaded.


cURL example:

$ curl \
  http://api.appcrafted.com/v0/assets/<ContainerID>/<AssetID>/upload_file/<AttrName> \
  -X POST -F file=@localfile.txt \
  -u <AccessKey>:

Upload an Image

POST /v0/assets/<ContainerID>/<AssetID>/upload_image/<AttrName>

Same as uploading a file, except this creates an attribute with Type IMAGE. This enables the Unity plugin to process the image as a texture

Get a single Asset

GET /v0/assets/<ContainerID>/<AssetID>

Retrieves a single asset.


Response:

{
  "Text": {
    "Type": "STRING",
    "Value": "Hello, world"
  },
  "Link": {
    "Type": "URL",
    "Value": "https://www.appcrafted.com"
  },
  "Image": {
    "Type": "IMAGE",
    "Value": "http://cdn.appcrafted.com/abc.png"
  },
  "AssetID": "example-asset",
  "ContainerID": "XYZ",
  "AssetCreatedDate": "2013-12-10T20:59:46.721Z"
}

cURL example:

$ curl http://api.appcrafted.com/v0/assets/<ContainerID>/<AssetID> -u <AccessKey>:

Get all Assets

GET /v0/assets/<ContainerID>/all

Retrieves all assets from the specified container.


Response:

Assets will be returned in an array named Assets. For example, this container holds one asset with three attributes (Text, Link, and Image). Note that Image holds a reference to the Appcrafted CDN.

{
  "message": "Success",
  "Count": 1,
  "Assets": [
    {
      "Text": {
        "Type": "STRING",
        "Value": "Hello, world"
      },
      "Link": {
        "Type": "URL",
        "Value": "https://www.appcrafted.com"
      },
      "Image": {
        "Type": "IMAGE",
        "Value": "http://cdn.appcrafted.com/abc.png"
      },
      "AssetID": "example-asset",
      "ContainerID": "XYZ",
      "AssetCreatedDate": "2013-12-10T20:59:46.721Z"
    }
  ]
}

cURL example:

$ curl http://api.appcrafted.com/v0/assets/<ContainerID>/all -u <AccessKey>:

Update an Asset

PUT /v0/assets/<ContainerID>/<AssetID>

For clients that don't support HTTP PUT:

POST /v0/assets/<ContainerID>/<AssetID>/update_post

Updates or deletes the attributes of the specified asset depending on the supplied JSON.


Update an attribute:

{
  <Attribute_Name>: {
    "Action": "UPDATE",
    "Type": <Type>,
    "Value": <Value>
  }
}

Delete an attribute:

{
  <Attribute_Name>: {
    "Action": "DELETE"
  }
}

Response:

{
  "message": "Asset Updated"
}

cURL example:

$ curl http://api.appcrafted.com/v0/assets/<ContainerID>/<AssetID> \
  -X PUT -H 'Content-Type: application/json' \
  -d '{"myAttribute": {"Action": "UPDATE", "Type": "NUMBER", "Value": 42}}' \
  -u <AccessKey>:

Delete an Asset

DELETE /v0/assets/<ContainerID>/<AssetID>

Deletes the specified asset.


Response:

{
  "message": "Asset Deleted"
}

cURL example:

$ curl http://api.appcrafted.com/v0/assets/<ContainerID>/<AssetID> \
  -X DELETE -u <AccessKey>:
